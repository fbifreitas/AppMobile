# DIRECTION CHANGE NOTICE

## Status

Ativo. Este documento substitui a narrativa anterior que posicionava o repositório como uma plataforma vertical de vistoria/avaliação/marketplace operacional.

## O que mudou

Antes:
- o repositório era narrado como plataforma white-label de vistorias
- o blueprint corporativo estava acoplado ao caso de vistoria
- o modelo canônico global refletia o fluxo do domínio inspection

Agora:
- o repositório é uma plataforma corporativa multi-domínio
- inspection continua como domain pack e tese estratégica
- wellness e church entram como domain packs adicionais
- o blueprint corporativo sobe de nível e deixa de ser sinônimo de inspection

## Motivo da mudança

A empresa passa a se posicionar como fornecedora de tecnologia, SaaS, network e marketplace para múltiplos mercados, aproveitando o core já construído.

## Efeito esperado

- documentação corporativa mais clara
- backlog separado entre plataforma e domínios
- melhor governança para agentes de código
- expansão de mercado sem acoplamento semântico à vistoria
