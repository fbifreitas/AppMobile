# LEGACY MIGRATION MAP

## Purpose

Mapear a relação entre a documentação antiga e a nova V2.

## Migration policy

### Active V2 docs
Devem substituir a narrativa ativa do repositório.

### Legacy docs
Devem ser mantidos quando:
- registram contexto histórico relevante
- explicam decisões de negócio passadas
- ainda ajudam no domínio inspection

## Recommended handling

- adicionar banner `LEGACY` nos documentos antigos mantidos
- mover gradualmente conteúdo inspection-specific para a pasta/domínio correto
- evitar manter dois documentos ativos dizendo coisas contraditórias

## Legacy classification example

### Become corporate V2 docs
- README de entrada
- blueprint ativo
- backlog ativo
- visão executiva ativa

### Become inspection legacy or inspection-specific
- materiais centrados em valuation/laudo/settlement como narrativa corporativa
- diagramas que apresentem inspection como sinônimo da empresa
