# ROADMAP CORPORATE AND DOMAINS

## Corporate roadmap

### Phase 1
- estabilizar o Platform Core
- separar inspection como domain pack explicito
- documentar tenant/white-label
- criar canonical model corporativo

### Phase 2
- habilitar wellness sobre core compartilhado
- habilitar church sobre core compartilhado
- consolidar shared foundations

### Phase 3
- consolidar dados e inteligencia cross-domain
- unificar observabilidade e metricas multi-tenant
- ampliar integracoes por vertical

## Domain roadmap

### Inspection
- preservar 4 ondas como roadmap do dominio
- fortalecer operacao, distribuicao e marketplace operacional
- evoluir backend/plataforma para orquestrar enrichment, OCR, fact reconciliation e publication de execution hints
- usar o vertical de real estate como especializacao inicial do incremento, sem contaminar o core com vocabulario especializado
- preparar retorno operacional do App Mobile como fonte primaria de evidencias para consolidacao progressiva do report

### Wellness
- agenda, sessao, presenca, billing e financeiro
- avaliar reaproveitamento futuro de capabilities horizontais como document ingestion, OCR, artifact storage e reconciliation quando aplicavel

### Church
- eventos, membros, doacoes, engajamento e operacoes comunitarias
- avaliar reaproveitamento futuro de capabilities horizontais como document ingestion, manual resolution, artifact trail e analytics-ready storage quando aplicavel

## Programa incremental ativo

O incremento atual deve ser tratado como programa corporativo habilitador, com primeira entrega forte dentro de `inspection`.

Capacidades corporativas/horizontais envolvidas:
- enrichment orchestration
- document OCR
- fact reconciliation
- confidence/sufficiency evaluation
- raw/normalized/curated storage
- analytics-ready data trail
- manual resolution workflow
- channel publication and channel return ingestion

Capacidades de dominio/especializacao inicial:
- smart configuration de check-in step 1
- smart configuration de check-in step 2
- smart configuration do menu da camera
- reconciliacao entre facts de backend, OCR e retorno de campo
- preparacao progressiva da base do laudo/report
