# Guia UML e Mapa de Implementação

## Como usar
- Use case: fronteira funcional
- Sequence: implementação de fluxos
- Class: modelo de domínio
- State: transições críticas
- Activity: processo operacional
- Component: responsabilidades por serviço
- Deployment: infraestrutura

## Regra
Funcionalidade crítica deve estar rastreada em:
1. caso de uso
2. sequência
3. estado ou atividade
4. domínio/classes
