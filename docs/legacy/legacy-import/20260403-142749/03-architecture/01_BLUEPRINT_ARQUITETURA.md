# Blueprint de Arquitetura

## Diretriz
Arquitetura orientada a domínio, distribuível desde o início, com serviços independentes nos contextos críticos.

## Modelo canônico
`Demand → Case → Job → Inspection → Valuation → Report → Settlement`

## Macrodomínios
### Ecosystem Core
- Identity & Access
- Tenant & Partner
- Integration Hub

### Marketplace + Execution Core
- Job Lifecycle
- Workforce & Dispatch
- Field Operations
- Process Orchestration
- Notification Service

### Technical + Financial Core
- Valuation Core
- Reporting & Sign-Off
- Pricing & Settlement
- Intelligence
